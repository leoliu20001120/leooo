## 一、项目概述

### 1.1 背景

- 新手玩家在海克斯大乱斗模式的留存率显著高于其他模式，但主动选择该模式的比例偏低。
- 推测核心障碍：前置引导不足，一局内需快速认知 10 个陌生英雄及大量海克斯符文，认知成本过高导致劝退。
- 海克斯选择是对局中**信息密度最高、决策压力最大**的环节，新手面对三选一时普遍存在"看不懂、不会选、怕选错"的困境。

### 1.2 产品定位

在对局过程中提供**轻量、及时、非强制**的 AI 教学伴侣，核心聚焦于**海克斯符文选择态**的智能辅助决策，同时覆盖地图机制引导和英雄快速认知。

### 1.3 目标用户

| 用户类型 | 特征 | 核心诉求 |
|----------|------|----------|
| 新手玩家 | 刚接触大乱斗，对海克斯机制不了解 | 快速理解符文效果，避免选错 |
| 回归玩家 | 有基础但版本断档，不了解新符文/新套装 | 快速补全版本知识 |
| 休闲玩家 | 不追求深度研究，希望轻松游戏 | 一键获取推荐，降低决策负担 |

### 1.4 核心目标

| 目标 | 衡量指标 | 目标值 |
|------|----------|--------|
| 降低上手门槛 | 新手海斗模式进入率 | 提升 15%+ |
| 提升决策质量 | 符文选择正确率（选择高胜率符文的比例） | 提升 20%+ |
| 提升留存 | 新手次日留存 / 7日留存 | 提升 5%+ |
| 不打扰体验 | 教练面板关闭/折叠率 | < 30% |

## 三、核心功能详细需求

### 3.1 海克斯选择态专属 UI

#### 3.1.1 整体布局

选择态为全屏覆盖弹层，AI 教练信息**内嵌在弹层中**（非背景层），避免被遮挡。

**布局层级（从上到下）：**

| 层级 | 区域 | 内容 |
|------|------|------|
| L0 | 顶部标题栏 | "海克斯强化符文选择" + 英雄信息 + 选择阶段（第 X/4 次） + 品质标识 |
| L1 | 卡片区 | 三张符文卡片并排 |
| L2 | 刷新提示行 | 剩余刷新次数 + 教练建议 |
| L3 | 教练推荐面板 | 推荐 + 理由 + 次选点评（可折叠） |
| L4 | 确认按钮 | "确认选择" |

#### 3.1.2 品质一致性规则

**核心规则：同一次选择中出现的三个海克斯必须为同一品质等级。**

| 品质 | 视觉表现 |
|------|----------|
| 银色 | 银灰色边框 + 银色条纹 + 银色图标底 |
| 金色 | 金色边框 + 金色条纹 + 金色图标底 |
| 棱彩 | 紫色渐变边框 + 彩虹条纹 + 紫色图标底 |

品质信息在标题栏的阶段徽章中展示，如："第 2/4 次选择 · 金色品质 · 已拥有：暴击律动"。

#### 3.1.3 符文卡片设计

参考游戏原生选择界面风格，每张卡片包含以下元素：

```
┌──────────────────────┐
│  [装饰角]  [?] [推荐] │  ← 左上装饰角；需二次理解符文右上显示 ? icon；推荐指数角标（A/B/C/D）
│                       │
│      ┌─────┐          │
│      │图标 │          │  ← 72px 圆形图标，品质色边框 + 光圈
│      └─────┘          │
│                       │
│      符文名称          │  ← 16px 加粗
│    [效果类型标签]      │  ← 圆角药丸标签，品质色底
│                       │
│   一句话效果描述       │  ← 12px 灰色文字
│                       │
│  [标签区：强力单卡     │  ← 可点击的功能标签
│   /黑科技/羁绊关联]   │
│                       │
│  刷新指数：12%         │  ← 该英雄下该符文的刷新概率
│       [ C ]           │  ← 刷新按钮，居中小方块
└──────────────────────┘
```

**? icon（二次理解标识）：**

对于效果描述不够直观、需要用户二次理解的符文（如"作弊我能回城"等改变基本游戏规则的符文），在卡片右上角（推荐指数角标左侧）显示一个 `?` icon。

| 项目 | 说明 |
|------|------|
| 触发条件 | 符文效果类型为"特殊机制"或人工标记为"需二次理解" |
| 交互 | 点击 `?` icon 弹出该符文的详细解读弹窗，包含白话解释 + 使用场景说明 |
| 视觉 | 20px 圆形半透明底，`?` 字符居中，品质色系 |
| 目的 | 降低新手对复杂/反直觉符文的理解门槛 |

**二次解释机制 — 完整规范：**

二次解释是 AI 教练的核心差异化能力之一。不同于简单的效果描述翻译，二次解释需要做到**"说人话 + 给场景 + 讲联动"**三层递进：

**第一层：白话翻译（What）**

将符文的游戏化描述翻译为日常语言，消除术语壁垒。

| 原始描述 | 白话翻译 |
|----------|----------|
| "获得回城能力" | "大乱斗本来不能回城买装备，选了这个就能像普通模式一样按B回城" |
| "获得一段位移并造成范围伤害" | "把你一个召唤师技能换成一个能往前冲的突进技能，冲到人堆里还能炸一圈" |
| "技能命中后触发装备特效" | "你每次丢技能打中人，都会额外触发装备的被动效果，相当于技能帮你多打了一下" |

**第二层：场景化说明（When & Where）**

说明这个符文在什么情况下最强/最弱，帮助新手建立判断框架。

| 符文     | 场景说明                                           |
| ------ | ---------------------------------------------- |
| 作弊我能回城 | "坦克英雄配心之钢最猛，因为可以反复回城叠属性；脆皮英雄拿了也行，死之前回城买装备比等死强" |
| 利刃华尔兹  | "近战刺客/战士拿了起飞，等于多一个突脸手段；射手/法师慎选，会失去闪现保命"        |
| 热身动作   | "站桩型法师（比如泽拉斯、维迦）蓄力后伤害很高；但大乱斗节奏快，很难安全站着不动"      |

**第三层：联动提示（Combo）**

如果该符文有明确的装备/符文联动关系，在二次解释中一并提示。

| 符文 | 联动提示 |
|------|----------|
| 暴击专精 | "每件暴击装额外 +10% 暴击伤害。你身上有几件暴击装就乘以几，出无尽+急速火炮+飓风时收益最大" |
| 头戴式装备大师 | "让你身上所有头部装备属性 +40%。头部装备主要是坦克装（兰顿、日炎等），射手/法师很少出" |
| 虚幻武器 | "技能能触发装备被动特效。法师出纳什之牙后，技能等于帮你打普攻特效；物理英雄配电刀也很猛" |

**二次解释触发规则：**

| 维度 | 规则 |
|------|------|
| **自动标记** | 效果类型为"🎲 特殊机制"的符文自动标记 |
| **人工标记** | 策划在符文知识库中手动标记 `needExtraExplain: true` |
| **数据辅助** | 如果某符文的"被刷新率极高但实际胜率不低"（即被误判为弱），自动建议标记 |
| **黑科技联动** | 具有黑科技组合关系的符文，即使不复杂也在二次解释中补充组合信息 |
| **装备类符文** | 涉及特定装备类型的符文（如"头戴式装备大师"），自动展示装备分类知识库 |

**解读弹窗结构：**

```
┌─ ❓ {符文名} — 详细解读 ───────────────── [✕] ─┐
│                                                   │
│  📖 白话解释                                      │
│  {第一层：白话翻译内容}                            │
│                                                   │
│  🎯 使用场景                                      │
│  {第二层：场景化说明}                              │
│                                                   │
│  🔗 联动提示（如有）                              │
│  {第三层：联动的装备/符文/黑科技}                  │
│                                                   │
│  💡 教练总结                                      │
│  "{一句话总结：适合谁 + 不适合谁}"                │
│                                                   │
└───────────────────────────────────────────────────┘
```

**卡片交互：**

| 操作 | 响应 |
|------|------|
| 点击卡片 | 选中，边框高亮，启用"确认选择"按钮 |
| Hover 卡片 | 教练面板切换为该符文的详细解读 |
| 离开卡片 | 教练面板恢复默认推荐内容 |
| 点击 `?` icon | 弹出详细解读弹窗（不影响卡片选中状态） |

**卡片视觉根据推荐指数变化：**

| 推荐指数 | 边框 | 特效 | 角标样式 |
|----------|------|------|----------|
| **S** | 品质色边框 + 金色光晕呼吸 + 粒子流光 | 金色渐变半透明叠层 + "天胡"文字标识 | 金色渐变底 + 闪光 |
| **A** | 品质色边框 + 光晕呼吸动效 | 金色渐变半透明叠层 | 金色渐变底 + 橙色文字 |
| **B** | 品质色边框 + 轻微光效 | 蓝色渐变半透明叠层 | 蓝色渐变底 |
| **C** | 品质色边框 | 无 | 白色半透明底 |
| **D** | 品质色边框（偏暗） | 无 | 深灰底 + 暗淡文字 |

### 3.2 推荐指数系统

#### 3.2.1 推荐指数定义（S/A/B/C/D 五级）

推荐指数基于**该符文在当前英雄上的胜率**（优先）或**全英雄平均胜率**（兜底），引入 S 级"天胡"概念，完整覆盖从"碰到就拿"到"不推荐"的全决策区间：

| 推荐指数 | 胜率区间 | 角标样式 | 教练话术模板 | 决策建议 |
|----------|----------|----------|-------------|----------|
| **S** | ≥ 55% | 金色渐变底 + 闪光呼吸动效 + "天胡"角标 | "碰到就拿，别犹豫" | 无脑选，效果碾压同级，不需要任何配合 |
| **A** | 52% ~ 55% | 金色渐变底 + 橙色文字 + 阴影光效 | "这个很稳，选它不亏" | 大多数情况都是好选择，直接推荐 |
| **B** | 49% ~ 52% | 蓝色渐变底 | "配你这个英雄挺合适" | 中等偏上，看英雄和已有符文的契合度 |
| **C** | 46% ~ 49% | 白色半透明底 | "中规中矩，没更好的可以拿" | 收益不突出，考虑刷新 |
| **D** | < 46% | 深灰底 + 暗淡文字 | "不太推荐，建议刷新" | 收益低或条件苛刻，优先刷新 |

> **S 级设计意图：** 在 A 级之上引入 S 级，用于区分那些"真正的版本答案"级别的符文。S 级符文出现概率低但影响力极大，需要在视觉和话术上给予最高辨识度，确保新手不会因为"看不出差别"而错过天胡。

#### 3.2.2 阶段化决策框架

符文推荐不仅取决于符文本身强度，还需考虑**选择所处的对局阶段**。同一符文在不同阶段的推荐权重不同：

| 选择节点 | 等级 | 品质倾向 | 决策重心 | 教练策略 |
|----------|------|----------|----------|----------|
| **开局选择** | Lv.1 | 银色/金色 | "万金油 > 一切"，优选简单易懂、不需配合的符文 | 优先推 S/A 级强力单卡；黑科技仅提示不强推 |
| **7级选择** | Lv.7 | 银色/金色 | "补强 or 成型"，可以开始考虑黑科技组合和羁绊凑套 | 如有黑科技可成型则优先推；否则推强力单卡 |
| **11级选择** | Lv.11 | 金色/棱彩 | "定型 or 转折"，基于已有符文/装备做联动推荐 | 重点分析已有符文组合，推联动收益最大的 |
| **15级选择** | Lv.15 | 金色/棱彩 | "补缺 or 决胜"，最后机会，需谨慎刷新 | 优先补短板；不建议冒险刷新；提示"最后一次" |

**阶段化推荐话术示例：**

| 阶段 | 场景 | 话术 |
|------|------|------|
| 开局 | S 级符文出现 | "开局就天胡了，碰到别犹豫直接拿！" |
| 开局 | 全 C/D 级 | "开局不急，三个都不强，大胆刷新碰碰运气。" |
| 7级 | 黑科技可成型 | "配上你开局拿的坦克引擎，不死坦克组合要成了！" |
| 11级 | 联动已有符文 | "你已有暴击律动+易损，拿暴击专精直接三件套成型。" |
| 15级 | A 级出现 | "最后一次选择了，A 级很稳，不建议赌刷新。" |
| 15级 | 全弱 | "最后一次了虽然都一般，挑一个最不亏的吧。" |

**"单卡强度 vs 羁绊"决策建议点：**

新手最常见的困惑是"这个符文强但不属于套装，那个弱但能凑套装"。教练需在此给出明确指引：

| 决策场景 | 推荐策略 | 教练话术 |
|----------|----------|----------|
| S/A 级强力单卡 vs 能凑套装的 C 级 | **推荐强力单卡** | "单卡够强就别凑套了，稳稳拿下比赌套装靠谱" |
| A 级单卡 vs 差 1 件激活套装的 B 级 | **推荐凑套装** | "你只差一件就激活套装了，套装收益比单卡更大" |
| B 级单卡 vs 差 2 件套装的 B 级 | **推荐单卡** | "套装还差两个太远了，先选眼前最强的" |
| 黑科技可成型 vs A 级单卡 | **视黑科技收益** | 黑科技组合胜率 > 单卡胜率 3%+ 则推黑科技 |

#### 3.2.3 数据源

| 数据项 | 维度 | 更新频率 | 说明 |
|--------|------|----------|------|
| 符文胜率 | 英雄 × 符文 | 每日 / 版本热更新 | 优先使用英雄维度，数据不足时回退全英雄平均 |
| 符文阶段胜率 | 英雄 × 符文 × 选择阶段 | 每日 | 区分开局/7级/11级/15级不同阶段的胜率表现 |
| 新符文冷启动 | 全英雄平均 | 上线首周回退，标注"新符文" | S/A 级评定暂时冻结 |
| 符文组合胜率 | 符文A × 符文B × 英雄 | 每日 | 两个符文共同出现时的联合胜率，用于黑科技判定 |

### 3.3 刷新机制

#### 3.3.1 核心规则

**每个符文拥有独立的一次刷新机会**，而非整体共用一次。即三选一时，每张卡片底部各有一个独立刷新按钮，玩家可以选择性地刷新任意一个或多个符文。

#### 3.3.2 刷新指数

**刷新指数**表示该英雄在大盘数据中对某个海克斯符文的刷新概率（即全服该英雄玩家刷新该符文的比例），直接展示在卡片底部刷新按钮上方，帮助玩家判断"这个符文值不值得刷"。

| 项目 | 说明 |
|------|------|
| 数据含义 | 全服该英雄玩家在遇到此符文时选择刷新的比例 |
| 展示格式 | `刷新指数：XX%`，显示在卡片标签区下方 |
| 数据源 | 英雄 × 符文 维度的被刷新率，每日更新 |
| 视觉规则 | 刷新指数 ≥ 40% 显示为橙色（暗示值得刷）；< 15% 显示为灰色（暗示不用刷）；其余为默认色 |

**刷新指数与刷新建议的关系：** 刷新指数为客观数据展示，刷新建议为 AI 教练的主观判断。两者互为参考但不完全对等。

#### 3.3.3 刷新按钮状态

| 状态 | 视觉 | 含义 |
|------|------|------|
| 建议刷新（suggest） | 橙色边框 + 呼吸光效动画 | 教练认为该符文较弱，建议尝试刷新 |
| 可选刷新（neutral） | 蓝灰色边框 | 可以刷新但非必要 |
| 不建议刷新（dimmed） | 深灰色，不可点击 | 该符文已经很强，不建议浪费刷新机会 |
| 已刷新 | 深灰色，显示"—"，不可点击 | 该符文的刷新机会已使用 |

#### 3.3.4 刷新建议逻辑

基于推荐指数和刷新指数综合判断：

| 推荐指数 | 刷新指数 | refreshAdvice | 教练话术 |
|----------|----------|---------------|----------|
| S | 任意 | dimmed（强烈不建议） | "天胡符文别刷，拿了就赚" |
| A 且热门 | < 15% | dimmed（不建议） | "这个很强，留着" |
| A | 15%~40% | neutral | — |
| B | 任意 | neutral | — |
| C | 任意 | suggest | "这个一般，可以试试刷新" |
| D | 任意 | suggest | "不推荐这个，建议刷新" |

**阶段修正：** 第 4 次选择（15级）时，刷新建议整体趋于保守：

| 阶段 | 调整规则 |
|------|----------|
| 15级 + 有 S/A 级 | 全部 dimmed，提示"最后一次了，稳一点" |
| 15级 + 最高仅 B 级 | suggest 降为 neutral，提示"最后一次了，B 级也能用" |
| 15级 + 全 C/D | 维持 suggest，但话术变为"最后一次了，碰碰运气也行" |

#### 3.3.5 刷新提示行

卡片区下方显示汇总信息：
- 剩余刷新次数：`X/3`
- 教练建议文案（如有建议刷新的符文则高亮提示）

#### 3.3.6 刷新后品质一致性

刷新替换出的新符文**必须与当前场景品质一致**。即金色品质场景刷新后仍为金色符文。

### 3.5 AI 教练推荐面板

#### 3.5.1 面板结构

```
┌─ 🤖 AI 教练推荐 ─────────────────── [▲ 折叠/展开] ─┐
│                                                      │
│  💡 推荐选【符文名】 — {一句话推荐理由}               │
│     "{教练详细点评}"                                  │
│                                                      │
│  · {次选1点评}                                       │
│  · {次选2点评}                                       │
│                                                      │
└──────────────────────────────────────────────────────┘
```

#### 3.5.2 推荐逻辑

推荐逻辑结合**推荐指数 + 阶段策略 + 组合联动**三重维度：

| 优先级 | 条件 | 推荐策略 | 适用阶段 |
|--------|------|----------|----------|
| 1 | 有推荐指数 S（天胡） | 直接推荐，最高优先级 | 全阶段 |
| 2 | 有推荐指数 A 且为强力单卡 | 直接推荐 | 全阶段 |
| 3 | 有黑科技可成型（联合胜率 > 单卡 3%+） | 优先推荐黑科技组件 | 7级起 |
| 4 | 有羁绊可激活（差 1 件） | 提醒羁绊凑套 | 7级起 |
| 5 | 有推荐指数 A（非强力单卡，需配合） | 推荐 A 级 + 附带配合说明 | 全阶段 |
| 6 | 有推荐指数 B | 推荐 B 级 | 全阶段 |
| 7 | 全为 C/D 级 | 建议逐个刷新 | 非15级 |
| 8 | 15级 + 全 C/D | 推荐其中最高的 + "最后一次，选最不亏的" | 仅15级 |

**推荐优先级冲突解决：**

当多个条件同时满足时（如既有 S 级强力单卡又有可成型黑科技），按以下规则裁决：

| 冲突场景 | 裁决规则 |
|----------|----------|
| S 级单卡 vs 黑科技 | S 级单卡优先（天胡不犹豫） |
| A 级单卡 vs 黑科技可成型 | 比较联合胜率 vs 单卡胜率，差值 > 3% 推黑科技 |
| A 级单卡 vs 差 1 件套装 | 如套装激活收益 > 2% 推套装，否则推单卡 |
| 黑科技 vs 套装 | 黑科技优先（收益通常更大且更直观） |

#### 3.5.3 交互行为

| 操作 | 响应 |
|------|------|
| Hover 卡片 | 面板内容切换为该符文的详细解读（效果类型 + 品质 + 评级 + 装备关联提示） |
| 离开卡片 | 面板恢复默认推荐 |
| 点击面板头部 | 折叠/展开面板 |
| 折叠状态 | 仅显示一行："🤖 AI 教练推荐 · 💡 推荐选【XX】" |

#### 3.5.4 刷新后更新

符文刷新后，教练面板**自动重新计算推荐**，推荐切换到新的最优符文。

### 3.6 地图/模式机制轻量讲解

#### 3.6.1 功能定位

在玩家进入对局的早期空档（出生/上线/等待阶段），以轻量气泡形式介绍当前地图或模式的核心机制，帮助新手快速建立基本认知，避免因不了解规则而劝退。

#### 3.6.2 输入

| 输入项 | 来源 | 说明 |
|--------|------|------|
| 当前地图 ID | 游戏客户端 | 嚎哭深渊、斗魂竞技场等 |
| 当前对局模式 | 游戏客户端 | 极地大乱斗(ARAM)、2v2v2v2 竞技场等 |
| 玩家是否首次进入该模式/地图 | 玩家历史数据 | 判断是否需要完整引导 |
| 当前阶段 | 游戏客户端 | 出生/上线/等待/非战斗阶段 |

#### 3.6.3 输出

| 输出项 | 格式 | 示例 |
|--------|------|------|
| 地图机制介绍 | 一句话，≤ 30 字 | "大乱斗只有一条路，不能回城，死了才能买装备" |
| 玩法提醒/推荐 | 一句话，≤ 30 字 | "地图中间有血包，记得抢" |

#### 3.6.4 触发条件

| 条件 | 说明 |
|------|------|
| 首次进入对应地图/模式 | 展示完整机制介绍（3~5 条核心规则） |
| 对局开始后一定时间窗口内 | 非首次进入时，展示 1~2 条进阶提醒 |
| 当前无更高优先级战斗提醒 | 避免与海克斯选择、危险预警等高优先级内容冲突 |

#### 3.6.5 内容覆盖

**a) 嚎哭深渊（极地大乱斗）**

| 机制 | 讲解内容 |
|------|----------|
| 单线战场 | 只有一条路，5v5 正面团战，没有打野和分线 |
| 不能回城 | 普通情况下无法回城，只有死亡后才能购买装备 |
| 随机英雄 | 系统随机分配英雄，可重随或与队友交换 |
| 血包机制 | 地图中央有生命圣物（血包），踩到回复生命值和法力值 |
| 海克斯符文 | 对局中 4 次海克斯选择，是大乱斗的核心策略点 |

**b) 斗魂竞技场（2v2v2v2）**

| 机制 | 讲解内容 |
|------|----------|
| 2v2 对决 | 8人分4组，每组2人，轮流2v2战斗 |
| 生命值系统 | 每组共享生命值，输掉一轮扣血，归零淘汰 |
| 购物阶段 | 每轮战斗前有购买和选择强化道具时间 |
| 海克斯增幅 | 每轮选择增幅效果，与搭档协调配合很关键 |
| 最终目标 | 存活到最后获胜，需平衡进攻和生存 |

#### 3.6.6 展示规则

| 规则 | 说明 |
|------|------|
| 首次进入 | 展示完整机制（3~5 条），附带 AI 教练提醒（2~3 条tips） |
| 非首次进入 | 展示 1~2 条进阶提醒，持续 8 秒自动收起 |
| 同一地图/模式仅首次完整展示 | 后续不再重复完整引导 |
| 玩家可手动关闭 | 提供关闭按钮，不强制阅读 |

#### 3.6.7 教练 Tips 示例

| 场景 | Tips |
|------|------|
| 嚎哭深渊 · 首次 | "开局不要急着出门，先看队伍阵容和对面英雄，想好出装方向再买" |
| 嚎哭深渊 · 首次 | "珍惜每次死亡复活的购买机会，攒金币一波到位比零散出装更强" |
| 嚎哭深渊 · 非首次 | "这把关注海克斯选择，AI教练会在每次出现时给你推荐" |
| 竞技场 · 首次 | "和搭档沟通阵容搭配，一人输出一人坦克/控制是最稳的组合" |

### 3.7 英雄认知辅助

#### 3.7.1 功能定位

当玩家在对局中首次遇到不认识的敌方英雄，或首次被某英雄显著影响体验时，AI 教练以轻量方式提供该英雄的定位/威胁说明和应对建议，帮助新手快速建立敌方英雄认知。

#### 3.7.2 输入

| 输入项 | 来源 | 说明 |
|--------|------|------|
| 当前局内英雄池信息 | 游戏客户端 | 双方 10 名英雄列表 |
| 玩家是否首次遇见该英雄 | 玩家历史数据 | 判断是否需要认知引导 |
| 玩家是否明显缺乏应对经验 | 数据分析（后续扩展） | 如被该英雄频繁击杀、被控时间过长等 |

#### 3.7.3 输出

| 输出项 | 格式 | 示例 |
|--------|------|------|
| 英雄定位/威胁说明 | 一句话，≤ 40 字 | "劫是高爆发刺客，大招标记你后几秒引爆巨额伤害" |
| 应对建议 | 一句话，≤ 40 字 | "他 R 你时马上贴墙站，出中娅沙漏能躲掉引爆伤害" |

#### 3.7.4 触发条件

| 条件 | 说明 |
|------|------|
| 首次遇见该英雄 | 在对局开始或上线阶段展示该英雄的威胁说明 |
| 首次被该英雄明显影响体验时 | 如被击杀、被长时间控制等（后续可扩展） |

#### 3.7.5 威胁等级

| 等级 | 判定依据 | 视觉标识 |
|------|----------|----------|
| 🔴 高威胁 | 该英雄对玩家当前英雄克制关系明显，或该英雄是高胜率刺客/爆发型 | 红色标识 |
| 🟡 中威胁 | 有一定威胁但可应对 | 黄色标识 |
| 🟢 低威胁 | 威胁较小或玩家英雄天然克制对方 | 绿色标识 |

#### 3.7.6 内容结构

每个敌方英雄的认知卡片包含：

| 字段 | 说明 |
|------|------|
| 英雄名称 + 头像 | 识别目标 |
| 定位描述 | 刺客/战士/法师/射手/辅助/坦克 |
| 威胁等级 | 高/中/低，附带颜色标识 |
| 是否首次遭遇 | 标记"🆕 首次遭遇" |
| 英雄定位/威胁说明 | 一段白话描述该英雄的核心能力和威胁点 |
| 应对建议 | 一段针对玩家当前英雄的具体应对策略 |

#### 3.7.7 AI 教练综合建议

当同局有多个高威胁敌方英雄时，教练额外提供综合建议：
- 优先关注哪个威胁
- 出装方向调整建议（如"优先出中娅沙漏"）
- 站位/打法提醒（如"站后排，别单独去吃血包"）

#### 3.7.8 内容示例

**场景：拉克丝 vs 劫**

| 项目 | 内容 |
|------|------|
| 威胁等级 | 🔴 高威胁 |
| 定位说明 | "劫是高爆发刺客，擅长瞬间秒杀脆皮。他的大招会标记你，几秒后引爆造成巨额伤害" |
| 应对建议 | "他 R 你的时候马上贴墙站，Q 技能朝他落地点丢。出中娅沙漏可以完美躲掉他的大招引爆伤害" |

**场景：金克丝 vs 布里茨**

| 项目 | 内容 |
|------|------|
| 威胁等级 | 🔴 高威胁 |
| 定位说明 | "机器人的Q技能（机械飞爪）是最恐怖的技能之一，命中直接把你拽到对面人堆里" |
| 应对建议 | "永远站在小兵后面，他的钩子会被小兵挡住。出边缘之夜能挡一次钩子" |

#### 3.7.9 展示规则

| 规则 | 说明 |
|------|------|
| 每个敌方英雄最多展示一次 | 避免重复打扰 |
| 优先展示高威胁英雄 | 低威胁可延后或省略 |
| 不打断海克斯选择 | 海克斯选择态期间不弹出英雄认知 |
| 可手动关闭 | 玩家随时可关闭认知提示 |
| 对局 2 分钟内展示 | 在玩家上线初期的空档展示 |

#### 3.7.10 数据需求

| 数据项 | 维度 | 更新频率 | 用途 |
|--------|------|----------|------|
| 英雄克制关系表 | 英雄 × 英雄 | 版本更新时 | 威胁等级判定 |
| 英雄定位标签 | 每个英雄 | 版本更新时 | 定位描述生成 |
| 英雄核心技能说明库 | 每个英雄的关键技能 | 版本更新时 | 威胁说明文本 |
| 英雄应对建议库 | 英雄 × 英雄 | 人工维护 + 数据辅助 | 应对建议文本 |
| 玩家英雄遭遇历史 | 玩家维度 | 实时 | 首次遭遇判定 |

### 3.8 教练语言风格规范

| 维度 | 要求 |
|------|------|
| 语气 | 轻松、像朋友聊天，不说教 |
| 长度 | 单条不超过 30 字（符文推荐可放宽至 40 字） |
| 表达 | 不给数值和百分比，用感受性描述（"很猛""特别适合""白送"） |
| 频率 | 宁可少说，不要刷屏；宁可晚说，不要抢关键操作时机 |
| 人称 | 用"你"而非"玩家""召唤师"，保持亲近感 |
| 新手友好 | 避免术语缩写（不说"CD"说"技能冷却"，不说"AD"说"物理攻击"），首次出现术语带白话解释 |

## 四、标签体系（后端推荐引擎）

驱动 AI 教练推荐逻辑的完整标签维度，**不直接暴露给用户**，由后端存储和计算。

### 4.1 维度一：效果类型

| 标签 | 含义 | 典型符文 |
|------|------|----------|
| 🗡️ 输出增幅 | 直接提升伤害 | 大力、玻璃大炮 |
| 🔁 技能加速 | 缩短技能冷却 | 循环往复 |
| 🛡️ 生存增强 | 提升坦度 | 坦克引擎、歌利亚巨人 |
| ❤️‍🩹 续航回复 | 持续治疗/吸血 | 吸血习性、作弊我能回城 |
| 💨 机动性 | 提升移速/位移 | 利刃华尔兹 |
| 🎯 攻击特效 | 普攻/技能附加效果 | 虚幻武器、台风 |
| 💥 爆发 | 短时间大量伤害 | 巨人杀手、易损 |
| 🧲 控制/干扰 | 减速/击退/控场 | 缩小射线 |
| 🤝 团队增益 | 增强队友 | 风语者的祝福 |
| 🎲 特殊机制 | 改变游戏基本规则 | 作弊我能回城 |
| 💰 经济收益 | 额外金币 | 当心小蛋糕 |

### 4.2 维度二：适用英雄类型

| 标签 | 适用谁 |
|------|--------|
| 近战适用 | 战士、刺客、坦克 |
| 远程适用 | 射手、法师、部分辅助 |
| 通用 | 所有英雄 |
| 法术英雄优先 | 靠技能打伤害的英雄 |
| 物理英雄优先 | 靠普攻打伤害的英雄 |
| 坦克/前排优先 | 站前面吸收伤害的英雄 |
| 辅助/增益优先 | 提供治疗护盾的英雄 |

### 4.3 维度三：装备联动

## 5.5 技术实现方案

### 5.5.1 整体架构

采用**离线预计算 + ES 检索 + 实时上下文拼装**的三层架构，确保在 500ms 内完成推荐计算：

```
┌─────────────────────────────────────────────────────────┐
│                   离线数据管线（每日/版本更新）             │
│                                                         │
│  大盘对局数据 ──→ Spark/Hive ──→ 「英雄×符文」胜率矩阵    │
│                         ├──→ 「英雄×符文」刷新率矩阵      │
│                         ├──→ 「符文×符文」组合胜率矩阵     │
│                         └──→ 「符文×符文×阶段」阶段胜率    │
│                                                         │
│  人工知识维护 ──→ 黑科技组合库 / 符文标签库 / 装备知识库    │
│                                                         │
│            ↓ 结构化写入                                   │
│                                                         │
│  ┌─────────────────────────────────────────┐            │
│  │         Elasticsearch 知识索引           │            │
│  │                                         │            │
│  │  Index: rune_profile                    │            │
│  │  Index: rune_combo                      │            │
│  │  Index: hero_rune_stats                 │            │
│  │  Index: equip_knowledge                 │            │
│  │  Index: hack_combo                      │            │
│  │  Index: rune_explain                    │            │
│  └─────────────────────────────────────────┘            │
└─────────────────────────────────────────────────────────┘
                         ↕ 实时查询
┌─────────────────────────────────────────────────────────┐
│                   推荐引擎服务（实时）                     │
│                                                         │
│  输入: heroId, candidateRuneIds[], ownedRuneIds[],      │
│        ownedEquipIds[], stage(1/7/11/15)                │
│                                                         │
│  Step1: ES 批量查询 hero_rune_stats (heroId × runeId)   │
│  Step2: ES 查询 hack_combo (候选 × 已有符文/装备)        │
│  Step3: ES 查询 rune_combo (候选互查 + 套装进度)         │
│  Step4: 本地规则引擎计算推荐指数、标签、刷新建议、推荐排序 │
│  Step5: 拼装教练话术模板                                  │
│                                                         │
│  输出: CoachRecommendation JSON                          │
│  耗时目标: < 200ms（ES round-trip < 50ms × 3 批次）      │
└─────────────────────────────────────────────────────────┘
```

### 5.5.3 推荐引擎核心流程

```
输入: heroId, candidateRunes[3], ownedRunes[], ownedEquips[], stage
  │
  ├─ Step1: 查 hero_rune_stats  → 获取3个候选符文的胜率/等级/刷新率
  │         query: { hero_id: heroId, rune_id: IN candidateRuneIds }
  │         → 计算推荐指数 S/A/B/C/D（按stage取阶段胜率）
  │
  ├─ Step2: 查 hack_combo       → 黑科技匹配
  │         query: { components.id: IN (candidateRunes + ownedRunes + ownedEquips) }
  │         → 筛选可成型/部分成型的组合
  │
  ├─ Step3: 查 rune_profile     → 获取标签/套装/装备联动
  │         → 计算羁绊进度
  │         → 判断是否需要二次解释
  │
  ├─ Step4: 本地规则引擎
  │   ├─ 按优先级排序推荐（S > 黑科技 > 套装 > A > B > 刷新）
  │   ├─ 按阶段调整策略（开局偏万金油 / 15级偏保守）
  │   ├─ 计算刷新建议（推荐指数 × 刷新指数 × 阶段修正）
  │   ├─ 生成标签列表（强力单卡/黑科技/羁绊，最多2个）
  │   └─ 解决冲突（单卡 vs 套装 vs 黑科技）
  │
  └─ Step5: 话术拼装
      ├─ 主推荐: 模板填充（"{grade}级推荐 + 教练点评"）
      ├─ 次选点评: 为非推荐的2个符文生成一句话说明
      ├─ 刷新提示: 汇总刷新建议
      └─ 输出 CoachRecommendation JSON
```

### 5.5.4 性能保障

| 环节 | 目标 | 方案 |
|------|------|------|
| ES 查询 | < 50ms/批次 | 使用 multi_search API 合并请求；hot/warm 分层，活跃版本数据全在 hot 节点 |
| 规则计算 | < 30ms | 纯本地内存计算，黑科技/套装规则编译为决策树 |
| 话术拼装 | < 20ms | 模板预编译 + 缓存 |
| 预加载 | 对局开始 | 根据玩家英雄预查询全部符文池数据到本地缓存 |
| 总耗时 | < 200ms | 三批 ES 查询可并行为两批（Step1 + Step2&3 并行） |

### 5.5.5 数据更新策略

| 数据类型 | 更新频率 | 更新方式 |
|----------|----------|----------|
| 胜率/刷新率/选取率 | 每日凌晨 | 离线 Spark 作业 → 全量写入 ES |
| 版本热更新 | 版本上线后 4h | 紧急回退到全英雄平均，48h 内用新版数据替换 |
| 黑科技组合库 | 版本更新 + 每周补充 | CMS 后台编辑 → 审核 → 同步 ES |
| 符文标签/属性 | 版本更新 | 游戏数据 API 同步 + 人工校准 |
| 二次解释内容 | 新符文上线时 | 策划编写 → 审核 → 写入 ES |
| 装备知识库 | 版本更新 | 游戏数据 API 同步 |

## 六、教练触发策略

### 6.1 发言优先级

| 优先级 | 内容类型 | 触发条件 |
|--------|----------|----------|
| P0 | 海克斯符文推荐 | 进入符文选择界面 |
| P1 | 危险预警 | 敌方关键技能/团战即将爆发 |
| P2 | 地图/模式机制讲解 | 首次进入对应地图/模式，对局开始后空档时间窗口内 |
| P3 | 英雄认知辅助 | 首次遇见敌方英雄，或首次被该英雄明显影响体验 |
| P4 | 己方英雄简介 | 加载完成、首次操作 |
| P5 | 套装组合提醒 | 二次及以上符文选择时 |

### 6.2 去重与冷却规则

- 同一条内容对同一玩家**仅触发一次**（基于玩家历史记录）
- 同一场对局，地图机制提示最多 **3 条**
- 教练连续发言间隔 ≥ **15 秒**
- 高优先级内容出现时，低优先级跳过不补发
