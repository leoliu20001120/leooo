## 核心结论

**项目定位**：景德镇制瓷历史模拟经营游戏，覆盖宋→元→明→清四幕，核心玩法 = **制瓷经营 + 科技树 + 剧情叙事**。当前处于中期开发阶段，设计方案与监修方（清华）基本对齐，4 月初向甲方设计办汇报。

---

## 一、历史设计方案（与监修对齐）

### 1.1 龙窑定位（重要争议已解决）
- **问题**：龙窑需山坡、与窑厂集中设定矛盾
- **历史逻辑**：
  - 宋代及以前 → "聚落式"生产，原料采矿、制坯、烧造就近完成
  - 元明时期 → 分工细化，制坯装饰向镇区集中
- **方案共识**：宋代第一幕采用家庭作坊式龙窑，放在原料产区沙盘（磁石矿附近），后续通过"解锁"体现产业转移

### 1.2 御窑呈现（3/19 汇报方案）
- **多维度体现**：订单系统（御窑类型订单）+ 科技节点（官搭民烧解锁）+ 剧情角色（督陶官互动）
- **成本控制**：2D 场景呈现御窑建筑，避免 3D 高成本
- **监修反馈**：清华老师无额外建议

### 1.3 产业分工体系（第四幕清代）
- 强化产业细分：订单流程自动化 + 沙盘建筑（坯房、窑房、红店、交草行、西金行）
- 布局逻辑：生产端靠东（近原料区），商贸端近江边（运输枢纽）
- 最小改动实现内容突出

### 1.4 关键历史元素
| 元素 | 定位 | 呈现方式 |
|------|------|----------|
| 墩子体系 | 原料标准化变革 | 采集阶段磁石设计为墩子模型，自持环节可见实体 |
| 过路窑 | 窑厂跨街扩张 | 城镇面貌更新 / 资产扩张玩法 |
| 釉下彩绘 | 科技树新增节点 | — |
| 釉上彩窑炉 | 科技树新增节点 | — |
| 官搭民烧 | 科技树新增节点 | — |

### 1.5 删除/规避内容
- ❌ 开片、窑变（非景德镇核心）
- ❌ 朝贡、清真寺（宗教审核风险）
- ✅ "月赛批" → 修正为"月成批"

### 1.6 主角服饰
- 两套：学徒/窑工共用（类汉服粗布衣）→ 民窑主（明代大髻装）
- 民窑体系，不涉及官服

## 四、技术 & 平台

### API 资源
- **目标**：V spice API（未就绪）
- **过渡**：暂用混元 API 练手和流程验证
- **风险**：切换时接口可能不同，需评估代码改动量（一周内评估）

### 小程序 vs 小游戏

- **风险**：需避免重度游戏化审核风险

## 六、CG 场景视觉方案

### 遗产点标识
- **问题**：高空鸟瞰下矿区/林区与普通地貌难区分
- **方案**：
  - 地形差异：瓷石矿 = 溪流切割纵向山谷 vs 林木区 = 汇水盆地山坳
  - 细节元素：河流飘松木、水车等动态场景
  - 地名标注

### 建筑空间布局
- 参考现状地图 + 现有图纸
- 布局逻辑：窑厂靠东近原料，交草行近江边
- 专家提供分布图和平面图纸
