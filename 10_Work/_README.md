# 💼 10_Work — 工作项目区

> 与工作直接相关的项目、业务分析、产品需求等内容。

## 目录组织原则

每个子文件夹代表一个**工作项目**或**业务领域**：

```
10_Work/
├── uc对局复盘/           # 对局数据分析与复盘
├── 海克斯大乱斗/          # AI教练产品设计
├── 新项目名称/            # 你的其他工作项目...
│   ├── _README.md        # 项目概述（推荐）
│   ├── notes/            # 笔记文档
│   ├── data/             # 数据文件
│   └── assets/           # 图片等资源
```

## 与 20_Study 的区分

| 放这里 ✅ | 放 20_Study ❌ |
|-----------|---------------|
| 工作项目的需求/方案/复盘 | 个人学习的技术笔记 |
| 业务数据分析报告 | 读书摘要、课程笔记 |
| PRD、设计文档 | 教程、编程练习 |
| 会议纪要、周报 | 认证考试准备 |

## 命名建议

- 文件夹名用项目名，简洁明了
- 每个项目建议放一个 `_README.md` 说明项目背景
- 数据文件（xlsx, csv）放在项目子目录下

## AI 约定

- AI 在此目录下工作时，应先阅读项目的 `_README.md`
- 生成的分析报告放在对应项目目录下
- 临时数据和脚本放 `.temp/` 目录，不要污染项目目录
