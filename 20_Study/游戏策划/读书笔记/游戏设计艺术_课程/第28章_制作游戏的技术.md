# 第28章 制作游戏的技术

> **板块**：协作与商业 | **课时**：28/34 | **对应透镜**：#102 技术
> **一句话总结**：**技术是四大元素之一**——理解技术的能力和限制，才能做出更好的设计。

---

## 🔑 核心概念

### 28.1 设计师需要懂技术吗？

**不需要精通编程，但需要理解**：
- 什么是**可能的**（技术能做什么）
- 什么是**昂贵的**（哪些功能消耗大量开发资源）
- 什么是**新兴的**（新技术带来什么新可能性）
- 什么是**危险的**（技术风险在哪里）

### 28.2 技术选型的考虑

| 考虑因素 | 问题 |
|---------|------|
| **平台** | PC？主机？手机？VR？多平台？ |
| **引擎** | Unity？UE？自研？Godot？ |
| **网络** | 单机？联网？实时？异步？ |
| **图形** | 2D？3D？画面质量级别？ |
| **团队能力** | 团队擅长什么技术栈？ |
| **预算** | 授权费？服务器费？ |

### 28.3 基础技术（Foundational Technologies）

Schell 强调一个概念——**基础技术**：

> 有些技术是"给定的"（大家都有），有些是"差异化的"（你独有的）。**竞争优势来自差异化技术。**

- 2000年代：3D引擎是差异化技术 → 现在是"给定的"
- 2010年代：物理引擎是差异化的 → 现在是"给定的"
- 2020年代：AI/PCG可能是差异化技术 → 未来可能变成"给定的"

### 28.4 技术与创新的关系

- **新技术 → 新玩法**：触屏 → 滑动游戏；VR → 沉浸体验
- **旧技术的新用法**：像素风不是技术倒退，而是美学选择
- **技术限制激发创意**：Game Boy的限制催生了《宝可梦》的设计精髓
