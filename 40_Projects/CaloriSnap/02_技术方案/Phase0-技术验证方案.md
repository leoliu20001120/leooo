# 🧪 Phase 0 — 技术验证方案

> 目标：用最小成本验证「拍照→识别标签→计算热量」的技术可行性
> 时间：2 周
> 输出：可行性验证报告 + OCR 方案选型结论

---

## 📋 验证清单

| # | 验证项 | 指标 | 达标线 |
|---|--------|------|--------|
| 1 | OCR 识别率 | 能正确提取品名+杯型+糖度 | ≥80% |
| 2 | LLM 结构化提取 | 从 OCR 文字中提取 JSON | ≥90% |
| 3 | 热量匹配准确率 | 匹配到正确饮品+计算合理 | ≥85% |
| 4 | 端到端延迟 | 拍照→结果展示总时间 | ≤5 秒 |
| 5 | 成本可控性 | 单次调用成本 | ≤0.05 元 |

---

## 🔬 OCR 方案对比

### 候选方案

| 方案 | 费用 | 精度预估 | 接入难度 | 适用场景 |
|------|------|---------|---------|---------|
| **微信 OCR（小程序内置）** | 免费 | ⭐⭐⭐ | 🟢 极简 | 小程序场景首选 |
| **腾讯云 通用印刷体 OCR** | 1000次/月免费，超出0.15元/次 | ⭐⭐⭐⭐ | 🟢 简单 | 精度优先 |
| **PaddleOCR（自部署）** | 免费（算力自担） | ⭐⭐⭐ | 🔴 需部署 | 成本敏感+大量级 |
| **多模态大模型直接识别** | ~0.01-0.03元/次 | ⭐⭐⭐⭐⭐ | 🟡 中等 | 省去结构化步骤 |

### 推荐验证路径

**路径 A（传统 OCR + LLM）：**
```
拍照 → 腾讯云 OCR → 提取原始文字 → LLM 结构化 → 匹配数据库
                                         ↓
                              { brand, drink, size, sugar, toppings }
```

**路径 B（多模态大模型一步到位）：**
```
拍照 → 多模态大模型（如 混元Vision / GPT-4o / Claude）→ 直接输出结构化JSON
                                                          ↓
                                              { brand, drink, size, sugar, toppings, 
                                                estimated_calories }
```

**建议：两条路径都测，对比效果和成本。**

## 🧪 测试脚本说明

### 脚本位置

```
CaloriSnap/
└── 02_技术方案/
    ├── Phase0-技术验证方案.md    ← 你在这里
    ├── ocr_test/
    │   ├── test_ocr.py          ← OCR 识别测试脚本
    │   ├── test_llm_extract.py  ← LLM 结构化提取脚本
    │   ├── test_e2e.py          ← 端到端测试（拍照→热量）
    │   ├── calori_calculator.py ← 热量计算引擎
    │   ├── drink_db.json        ← 结构化饮品数据库（种子数据）
    │   ├── requirements.txt     ← 依赖
    │   ├── sample_labels/       ← 存放测试标签照片
    │   └── results/             ← 测试结果输出
    └── 热量数据库-种子数据.md
```

### 运行方式

```bash
# 1. 安装依赖
cd 02_技术方案/ocr_test
pip install -r requirements.txt

# 2. 设置 API Key（选一个）
export TENCENT_SECRET_ID="your_id"
export TENCENT_SECRET_KEY="your_key"
# 或
export OPENAI_API_KEY="your_key"

# 3. 测试 OCR 识别
python test_ocr.py --image sample_labels/test1.jpg

# 4. 测试 LLM 结构化提取
python test_llm_extract.py --ocr-text "生椰拿铁 大杯 半糖 +珍珠"

# 5. 端到端测试
python test_e2e.py --image sample_labels/test1.jpg

# 6. 批量测试
python test_e2e.py --dir sample_labels/ --report results/report.json
```

## ✅ 成功标准

Phase 0 通过的条件：

1. **OCR 识别率 ≥ 80%**：主流品牌的打印标签可以稳定识别
2. **端到端准确率 ≥ 70%**：从照片到热量结果，70% 的情况可以自动完成
3. **成本可控**：单次识别成本 < 0.05 元
4. **延迟可接受**：端到端 < 5 秒

如果达标 → 进入 Phase 1 开发 MVP
如果不达标 → 评估是优化 OCR 还是换方案（如纯手动输入+搜索）
