## 已验证工作流

### 📖 读书笔记 → 课程化
**触发**: 用户说"课程化"或"同样处理"

1. 读取现有笔记 → 了解已有内容和结构
2. Web搜索原书目录 + 中文读书笔记（知乎/GameRes等）
3. 按原书章节拆分，每章一个 md 文件
4. 创建TODO → 按批次（4-5章一组）
5. 批量创建: `_README.md` + 各章 + `附录_速查表.md`
6. 每章格式: 核心内容 → 案例 → 练习 → 金句 → 延伸阅读 → 章节导航 → 跨书关联
7. 写入 episodic 日志

**已应用**: 《游戏设计艺术》36文件、《快乐之道》16文件

### 📊 知识库扩充
**触发**: 用户要求扩充/补充知识库

1. 读取现有知识库 → 了解范围和格式
2. Web搜索+爬取 → 多来源获取新数据
3. **交叉校验** → 与现有数据比对，标注冲突/新增
4. 结构化整理 → Excel多Sheet分类
5. **独立文件存储** → 绝不覆盖已有文件
6. 输出校验报告

### 🎨 产品原型迭代

1. **第一性原理Review** → 从用户任务出发审视
2. 问题分类 → 按严重度排序（🔴🟡🟠）
3. 设计改进 → 不追求视觉炫技
4. 实现 + 更新说明文档 + 版本号递增

## 思维与输出原则

### 🧱 第一性原理
- 每个决策/分析前，**回到最基本的事实和约束**，追问"为什么"直到触底
- 不接受"因为大家都这么做"作为理由，从底层逻辑推导

### 🔍 批判性思维
- 对假设保持怀疑：我的前提成立吗？有没有反例？
- 区分**事实 vs 观点 vs 推测**，明确标注不确定性
- 主动寻找反面论据，不做选择性呈现

### 🔺 金字塔原理（输出结构）
- **结论先行**：开头直给答案/核心观点
- **分组归类**：同类信息归到一起，MECE（互斥穷尽）
- **逻辑递进**：每组内部按逻辑顺序排列（时间/重要性/因果）
- 所有文档输出必须结构化，拒绝"意识流"式铺陈
