## CaloriSnap v0.3 — 第一性原理 Review + 全面重构

用第一性原理和批判性思维 review 了 v0.2 原型，发现 6 类问题后全面重构升级到 v0.3。

### Review 发现的核心问题

| # | 问题 | 严重程度 | 本质原因 |
|---|------|---------|---------|
| 1 | 首页优先级错位：CTA 藏在 tab 底部 | 🔴 高 | 信息架构未以用户核心任务（拍照）为中心 |
| 2 | 结果页信息过载：公式/系数用户不关心 | 🟡 中 | 产品视角 vs 用户视角混淆 |
| 3 | 无记录成功反馈，用户不确定是否保存 | 🔴 高 | 缺少操作确认的基本 UX 原则 |
| 4 | 无空状态/失败状态 | 🟡 中 | 只考虑了 happy path |
| 5 | 数据可视化不可读（柱状图无数值） | 🟡 中 | 装饰性 > 功能性 |
| 6 | 环形仪表盘空间效率低 | 🟠 低 | 视觉炫技 > 信息密度 |

### v0.3 改动清单

**信息架构重构：**
- 首页 hero 从纯装饰渐变改为「快速拍照 CTA 卡片」，用户一眼看到核心功能
- 仪表盘从 SVG 环形图改为水平进度条，信息密度更高、空间更省
- header 从渐变背景改为干净的白底 + 日期 badge
- 动态 greeting 根据时间自动切换（早上好/下午好/晚上好）

**结果页简化：**
- 删除「计算明细」公式块（×杯型系数、×温度系数 等用户不关心的细节）
- 替换为「规格标签条 + 三列影响摘要」（基础热量/加糖/咖啡因）
- 新增「预算影响条」— 可视化展示记入后预算使用情况
- CTA 从左右并列改为上下排列，主按钮（记入日志）在上、次按钮（再拍一杯）在下

**状态完善：**
- 新增识别失败页（`fail-overlay`）— 三个选项：重新拍照 / 手动输入 / 返回首页
- 新增空状态（`empty-state`）— 新用户/当天无记录时显示引导拍照
- 新增记录成功动画（`success-overlay`）— ✓ 绿色勾 + 剩余预算提示
- 新增 Toast 消息系统 — 各种操作反馈
- 拍照页新增"手动输入"入口

**微交互：**
- 所有数字使用计数器动画（ease-out cubic，数字从 0 跳到目标值）
- 进度条加载动画（width 过渡）
- Tab bar 拍照按钮脉冲呼吸光环提示
- 柱状图增加数值标签（hover/今天自动显示）
- 空数据日（周二/周日）用扁平灰色最小高度，不再显示 3% 假数据

**视觉优化：**
- 版本 badge 居中 + 低调半透明（不再抢眼金色）
- 配色保持「茶咖双生」不变，整体更克制
- 我的页增加「切换空状态演示」入口方便 stakeholder 预览
