## 工作与学习的区分

```
你的某次活动
    ↓
是跟公司/业务直接相关的吗？
    ├── 是 → 10_Work/
    └── 否 → 是自我提升/兴趣学习吗？
                ├── 是 → 20_Study/
                └── 否 → 是个人产品/Side Project？
                            ├── 是 → 40_Projects/
                            └── 否 → 是可复用的通用知识？
                                        ├── 是 → 30_Common/
                                        └── 不确定 → 00_Inbox/（先放着，以后再归档）
```

## 约定

### 🧑 给人类
- 新笔记先放 `00_Inbox`，定期归档
- **工作相关** → `10_Work`，**个人学习** → `20_Study`
- 每个项目目录建议放一个 `_README.md`
- 用 `[[双链]]` 关联笔记，构建知识网络
- 从工作和学习中提炼的**可复用经验**沉淀到 `30_Common`

### 🤖 给 AI
- 进入任何目录前，优先读取该目录下的 `_README.md`
- 临时文件和中间数据写入 `.temp/` 对应子目录
- 不要修改 `99_Templates/` 下的模板文件
- 生成的分析报告放在对应项目目录下
- 区分工作和学习内容，归档到正确的目录
- 个人产品项目放 `40_Projects/`，与工作项目 `10_Work/` 严格分离
